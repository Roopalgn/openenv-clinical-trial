## Charts Checklist

All charts should be generated by `plot_rewards.py` and saved to `results/`.

### Required for Pitch (3-minute demo)

- [ ] **Reward curve** — per-episode scatter + 20-episode rolling average + trend line with slope annotation. File: `results/reward_curve.png`
- [ ] **Before/after comparison bar** — Random vs Scripted vs Trained success rates at Warmup. File: `results/comparison_bar.png`
- [ ] **Capability radar** — 6-axis spider chart (phase compliance, FDA, info gathering, calibration, efficiency, adaptability) with 3 overlaid polygons. File: `results/capability_radar.png`
- [ ] **Curriculum timeline** — horizontal bar showing episodes at each tier, color-coded. File: `results/curriculum_timeline.png`

### Required for Mini-Blog

- [ ] **Architecture diagram** — from `ARCHITECTURE.md`. File: `results/architecture_diagram.png` (screenshot of ASCII or generated)
- [ ] **Reward component trends** — stacked area chart or multi-line showing `r_validity`, `r_ordering`, `r_info_gain`, `r_terminal_success` over episodes. File: `results/component_trends.png`
- [ ] **Scenario breakdown heatmap** — 4 scenarios × 5 tiers success rate grid. File: `results/scenario_heatmap.png`

### Nice-to-Have for Dashboard

- [ ] **Action diversity heatmap** — action types × episode windows, cell intensity = usage frequency. File: `results/action_heatmap.png`
- [ ] **Milestone velocity chart** — per-episode milestone completion time vs tier targets. File: `results/milestone_velocity.png`
- [ ] **Best episode replay** — step-by-step with phase colors and reward waterfall. File: `results/best_episode_replay.png`
